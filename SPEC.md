## プロジェクト概要

- **MOD名**: McHeliNavalAddon
- **対象**: Minecraft 1.14.4 / Forge
- **言語**: Java 8
- **依存MOD**: MCHELI（必須・リフレクション経由でアクセス）
- **MCHELIバージョン**: 1.14
- **設計方針**: 自律飛行なし。プレイヤーが手動操縦する前提の甲板装備をブロックとして追加する。モデリング不要。

## F1: カタパルト

### 概要
ブロック上の機体を前方へ強制加速して打ち出す。

### 動作仕様
- カタパルトブロックを複数並べてレール状に設置
- 機体がブロック上にいる状態で右クリックでアクティベート
- ブロックのfacing方向へ機体の速度を強制設定
- 打ち出し速度は `config/mchelinaval.toml` で変更可能（デフォルト: TBD ※MCHELI解析後）
- 発射時にスチームパーティクル＋効果音
- 発射と連動して隣接する移動プラットフォームに「機体セット」「機体離脱」を通知（JBD用途）

### 受け入れ条件
- [ ] 右クリックで機体が前方へ打ち出される
- [ ] 打ち出し方向がブロックの向きに従う
- [ ] 速度値をconfigで変更できる
- [ ] スチームエフェクトが出る

## F2: 移動プラットフォーム（UGOCRAFT式）

### 概要
コントローラーブロック1個で、プレイヤーが建築した既存ブロック構造を丸ごと動かす汎用システム。
モデリング不要。バニラブロックや他MODのブロックをそのまま使える。

### 動作仕様

**構成**
- 移動コントローラーブロック（1個）を構造物に設置
- 動かしたいブロックを何らかの方法で選択・登録
- 右クリックで移動開始・停止トグル

**移動方向**
| 方向 | 内容 | 用途 |
|---|---|---|
| Y軸（縦）| 上下に移動 | デッキエレベーター |
| Y + Z 同時（斜め）| 斜め45°方向に移動 | JBD（ジェットブラストデフレクター）|

**JBD用途での動作シーケンス**
```
① 機体がカタパルトブロックに乗る   → コントローラーが「展開」方向へ移動開始
② 機体がカタパルトから離れる        → コントローラーが「格納」方向へ移動開始（元の位置へ戻る）
```
カタパルトと隣接していればこの連動が自動で有効になる。

**設定項目**
- 移動方向（Y / Y+Z斜め）
- 移動距離（ブロック数）
- 移動速度（毎tickの移動量、config変更可）
- 対象範囲（コントローラーから何ブロック以内を動かすか）
- トリガー種別（手動 / カタパルト連動）

**エンティティ追従**
- 移動対象ブロック上にいる機体・プレイヤーも一緒に移動する

### 使用例
| 用途 | 移動方向 | トリガー |
|---|---|---|
| デッキエレベーター | Y（縦）| 手動 |
| JBD | Y + Z（斜め）| カタパルト連動 |

### 受け入れ条件
- [ ] コントローラーを右クリックで登録した構造物が動く
- [ ] 機体・プレイヤーも一緒に移動する
- [ ] 斜め移動（Y+Z同時）で45°展開に近い動きができる
- [ ] カタパルト連動モードで機体セット・離脱に応じて自動開閉する

## 非機能要件

| 項目 | 内容 |
|---|---|
| パフォーマンス | 移動プラットフォームの処理は10tick毎に間引く |
| モデリング不要 | カスタム3Dモデルを一切使わない。バニラブロックで完結 |
| 設定ファイル | `config/mchelinaval.toml`（Forge 1.14.4標準形式）|
| MCHELIアクセス | リフレクション経由。try-catchで保護。null安全徹底 |
| サーバー/クライアント | サーバーサイド処理主体。エフェクトはクライアント同期 |

## MCHELI解析が必要な事項（実装前に確認）

| # | 確認項目 | 用途 |
|---|---|---|
| 1 | 機体エンティティの基底クラス名 | フック対象の特定 |
| 2 | 速度フィールド名・型（XYZ成分か単一スカラーか）| カタパルトの実装 |
| 3 | 機体がブロック上にいるかの判定方法 | カタパルト発火トリガー・JBD連動 |
| 4 | 速度書き換え後に挙動が変わるか | 実装可否の確認 |
